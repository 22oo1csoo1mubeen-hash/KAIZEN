# KAIZEN - Project Flow

The UI design has been finalized.

The attached reference images are the single source of truth.

Do NOT redesign, reinterpret or modernize any section.

Your job is implementation only.

Every implementation must preserve:

- Layout
- Visual hierarchy
- Typography
- Spacing
- Glassmorphism
- Gradients
- Glow effects
- Shadows
- Border radius
- Component positions
- Colors
- Premium aesthetic

Implementation priority:

1. Pixel-perfect recreation.
2. Clean React architecture.
3. Reusable components.
4. Responsive behaviour.
5. Smooth Framer Motion animations.

Project Structure

Landing Page
├── Hero
├── Dashboard Showcase
├── Core Features (Habits, Goals, Expenses)
├── Why Kaizen
└── Footer

Workflow

Implement only ONE section at a time.

Never modify completed sections unless explicitly instructed.

Each completed section becomes locked.

Use:

- React
- TypeScript
- Tailwind CSS
- Framer Motion
- Lucide React

Use custom assets when provided.

Do not replace them with generated alternatives.

Do not invent new layouts.

Do not invent new typography.

Do not invent new spacing.

If any visual element cannot be reproduced exactly using CSS, recreate it as closely as possible while preserving the overall appearance.

The objective is to make the final website visually indistinguishable from the approved reference images.
